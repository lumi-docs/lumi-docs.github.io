# Setting up your console

This is an introduction to Google Cloud Platform (GCP). In this tutorial, we will create a GCP account and familiarize ourselves with the console and some basic components.

# Display GCP console

  1. Open an incognito window on Chrome web browser or an equivalent private window on any other browser. We use an incognito window so that your Google account for cloud services, does not interfere with your personal or any other Google account, you might be logged into.
  2. Go to https://console.cloud.google.com
  3. Google sign-in page will appear. If you have a Google account that was created using your work email, sign-in using that account. Otherwise, click on the Create account link and follow the instructions to create an account using your work email. Use this newly created account to sign-in.
  4. Click through any popups or instructions to reach the console window (see below).

5. Click on this button at the top left of the webpage to open the Navigation Menu. The menu will appear along the left edge of the screen. All GCP resources can be accessed through this menu so we will be using it a lot.

# Create a project and activate billing

  1. Click on this dropdown at the top left of the webpage.
  2. Click NEW PROJECT on the popup that appears.
  3. Enter a project name on the next window and click CREATE.
  4. The dashboard of your newly create project will appear on the console.
  5. Observe the different components on the dashboard.
  6. Note that your project name appears on the dropdown from step 1.
  7. The dashboard of your current project (project you have been working on) will appear every time you open the GCP console.
  8. Next, click the activate button at the top right of console.
  9. Go through the instructions to start your free trial and activate billing.
  10. If you cannot see the activate button for some reason, please go to the billing section on the navigation menu to activate billing.

# A quick tour

Open the navigation menu (click on console) and find all the resources available on GCP. The resources are divided into various categories. Two quintessentials of a cloud environment are COMPUTE and STORAGE resources. As the terms suggest, COMPUTE refers to the computation solutions and STORAGE refers to the storage solutions that might be suitable for a project.
Let's go through a few of the resources from our navigation menu.

  1. Compute Engine: Compute engine is a tool on GCP used to spawn virtual machine instances. It lets you create custom virtual machines on Google's Infrastructure.
  2. Storage: Google Storage uses buckets to store data. A bucket is like a folder on a virtual drive with infinite space. Buckets are associated to a project and you can have multiple buckets in a project. If a project is deleted, the buckets associated with it and the data within, are also deleted.
  3. ML engine: Machine Learning engine is a managed service that lets you run ML models on Google Cloud. Using ML engine, you can train a model, deploy it on cloud and draw predictions using an API. In the next tutorial, we will be training a locally created model on ML engine.
