# M series JSON data

Last updated: Mar 21, 2019
Read time: 3 mins

Here you will find the lastest information about the migrated M series data and instructions to access it.

# Instructions

The M-Series migrated data is divided into two partitions: old data (prior to 2017) and new data. There are 3 folders within each partition(folder): jsons, processed_data and raw_data. Please refer to the Folder structure section below. The JSON files present in the jsons folder should be the starting point for any exploration. Each JSON file corresponds to an M-series image and contains all the metadata regarding that image including the links to the actual binary data. A python script named read_data.py is present in the main folder. Please import this file in you python script and use the functions provided in it to conduct basic tasks such as viewing a JSON file, reading a JSON file into a python dictionary, loading the binary data into numpy arrays and accessing any other field in the JSON file. Please read more about these functions in the Reading Data section below.

Latest version of all data can be found at W:\M_Series_migration

# Folder structure

# Reading Data

The python script, read_data.py contains some functions/methods to perform basic tasks. These are as follows:

# view_json(json_path):

  • Description:
    Pretty print a JSON file.
  • Input:
    json_path | string | Full path to the JSON file
  • Returns:
    Nothing (prints JSON on console)

# read_json(json_path):

  • Description:
    Load a JSON file into a python dictionary.
  • Input:
    json_path | string | Full path to the JSON file
  • Returns:
    A python dictionary

# get_data(json_path, which_data):

  • Description:
    Load raw image, PD image or processed image (planes and mask array).
  • Inputs:
    json_path | string | Full path to the JSON file
    which_data | string | 'raw' for raw image, 'pd' for PD image and 'proc' for processed data(planes and mask)
  • Returns:
    A numpy nd array

# get_field(json_path, field):

  • Description:
    Load any field from the json. Note: This is a recursive function.
  • Inputs:
    json_path | string | Full path to the JSON file
    field | string | The key you want to search. Ex. 'calibration_data', 'sensor_config', 'finger_id', etc.
  • Returns:
    A variable of any of the following types : string, int, float, dict, list
